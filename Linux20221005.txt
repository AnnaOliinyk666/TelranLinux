Укажите след. команды:



для полной очистки индекса git stash apply с опцией --index

для просмотра разницы между раб.каталогом и посл. коммитом git show

для создания новой ветки и одновременного переключения в на нее git checkout -b

для сравнения веток между собой git diff

для удаления локальной ветки git branch -d

для слияния веток git merge

для перезаписи посл/коммита. git commit--amend


Коротко опишите причину возникновения конфликтов при слиянии и порядок действий для их решения.

Если разработчик A попытается изменить код, который редактирует разработчик B, может произойти конфликт. 
Для предотвращения конфликтов разработчики работают в отдельных изолированных ветках. 
Основная задача команды git merge заключается в слиянии отдельных веток и разрешении любых конфликтующих правок.
Локальное состояние необходимо стабилизировать с помощью команд git stash, git checkout, git commit или git reset.
